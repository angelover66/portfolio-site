## 二、技术架构详图

```
用户浏览器
    │
    ▼
Vercel CDN (SSG静态文件)
    │
    ▼
Next.js App Router
    ├── layout.tsx ── 全局布局 + 字体 + Metadata
    └── page.tsx ── 单页主入口
         ├── Navbar ── 固定顶部导航
         ├── Hero ── 个人介绍首屏
         ├── ProjectCard[] ── 项目卡片网格
         │    └── onClick → ProjectModal ── 详情弹窗
         │         ├── 常规项目: liveUrl + githubUrl 按钮
         │         └── 小程序项目: QR code 图片
         └── Footer ── 页脚
```

**数据流向**：`data/projects.ts` → `page.tsx` (map) → `ProjectCard` (展示) + `ProjectModal` (详情)

## 三、核心技术决策

### 3.1 纯静态 vs CMS

| 候选方案 | 优点 | 缺点 |
|---------|------|------|
| 纯静态硬编码 | 零依赖、部署快、改动即上线 | 每次更新需改代码 |
| Headless CMS | 非技术人员可编辑 | 增加复杂度和成本 |

**选择**：纯静态。5个项目的个人作品集更新频率极低（每月2-3次），CMS的富文本编辑优势不成立，反而增加维护负担。

### 3.2 多种项目入口的条件渲染策略

不同项目有不同访问方式：
- Web应用 → 在线链接 + GitHub
- 微信小程序 → 二维码扫码
- 仅有GitHub的CLI工具 → 仅GitHub按钮

**设计决策**：在 Project 接口增加可选 `qrcode` 字段，ProjectModal 采用"二维码优先"分支逻辑：
- 有 `qrcode` → 显示二维码 + 提示文案
- 无 `qrcode` → 按 `liveUrl` / `githubUrl` 分别渲染按钮
- 两项都无 → 隐藏整个操作区域

### 3.3 Tailwind v4 选择

create-next-app 默认安装 Tailwind v4，采用 CSS-based 配置（@theme 指令），不再需要 tailwind.config.ts。部分 opacity 修饰符在 v4 中行为不同，改用 inline style 兜底。

## 四、数据结构设计

### 4.1 Project 接口

```typescript
interface Project {
  id: string;            // 唯一标识，用于 key
  name: string;          // 项目名称
  tagline: string;       // 一句话定位
  description: string[]; // 2-3段详细描述
  techTags: string[];    // 技术标签
  liveUrl: string;       // 在线体验链接，无则为 ""
  githubUrl: string;     // GitHub链接，无则为 ""
  emoji: string;         // 项目emoji图标
  qrcode?: string;       // 可选，小程序二维码路径
}
```

### 4.2 状态机定义（弹窗）

```
CLOSED ──onClick(project)──▶ OPEN ──onClose/ESC/点击遮罩──▶ CLOSED
                                │
                                └── 打开时 document.body.style.overflow = "hidden"
```

## 五、组件树与路由

### 5.1 路由

单页应用，无路由切换。所有内容在 `/` 渲染。

### 5.2 组件层级

```
layout.tsx
  └── <html lang="zh-CN">
       └── <body>
            └── {children} → page.tsx
                 ├── <Navbar />
                 ├── <main>
                 │    ├── <Hero />
                 │    ├── <section> 项目卡片区
                 │    │    └── projects.map → <ProjectCard />
                 │    └── <section> Footer
                 │         └── <Footer />
                 └── {selectedProject && <ProjectModal />}
```

### 5.3 状态管理

无需全局状态管理库。弹窗状态通过 `page.tsx` 中的 `useState<Project | null>` 管理，向下传递 `onClick` 回调和当前选中 project。

## 六、边界与异常处理

| 场景 | 处理方式 |
|------|---------|
| 弹窗打开 | 禁止背景滚动（overflow: hidden），恢复时还原 |
| ESC键 | window.addEventListener 监听，关闭弹窗 |
| 点击遮罩 | onClick={onClose} 在遮罩层，内部 stopPropagation |
| 外部链接安全 | target="_blank" + rel="noopener noreferrer" |
| 空 liveUrl | 条件渲染，不显示按钮 |
| 空 githubUrl | 条件渲染，不显示按钮 |
| 有空qrcode | 显示二维码替代按钮区域 |
| 所有链接为空 | 隐藏整个操作按钮行 |
