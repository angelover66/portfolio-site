# 个人AI作品集网站 — 设计文档

> 版本：v1.0 | 日期：2026-05-19 | 状态：已确认

---

## 一、产品概述

### 1.1 产品定位
个人 AI 项目作品集展示网站，面向面试官和同行，展示 AI 产品经理 + AI Agent 开发者的能力。

### 1.2 核心目标
- 展示 3 个 AI Agent 项目的基本信息和在线访问入口
- 传达个人品牌：AI 产品经理 × AI Agent 开发者
- 作为面试素材和作品集沉淀

### 1.3 访问者路径
进入网站 → Hero 区了解身份 → 浏览项目卡片 → 点击卡片查看详情弹窗 → 点击链接访问项目/GitHub

---

## 二、技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 框架 | Next.js (App Router) | 专业感、SSG 静态生成、Vercel 原生支持 |
| 样式 | Tailwind CSS | 快速开发、渐变和毛玻璃效果支持好 |
| 动效 | Framer Motion | React 动效标准方案、弹窗缩放动画 |
| 部署 | Vercel (GitHub 自动部署) | 免费、推送即部署、全球 CDN |
| 字体 | Inter (英文) + 系统中文 | 现代几何感 + 零加载成本 |

---

## 三、页面结构

单页应用，4 个区块自上而下：

### 3.1 Navigation（顶部固定）
- 左侧：姓名 "Lulu Yang"
- 右侧：GitHub 图标 + Email 图标（hover 显示地址）

### 3.2 Hero（首屏核心区）
- 大标题：姓名
- 身份标签：「AI 产品经理 × AI Agent 开发者」
- 一句话自我介绍（1-2 行）
- 视觉：流动渐变背景 + 文字淡入动画

### 3.3 项目卡片区
- 标题："My AI Projects"
- 3 张卡片横向排列（移动端纵向堆叠）
- 每张卡片：毛玻璃效果 + 项目名称 + 一句话描述 + 技术标签 + "查看详情 →" 触发
- 点击卡片 → 弹窗展示详情

### 3.4 Footer
- 简短版权信息 + 联系方式

## 四、弹窗设计

### 4.1 触发方式
点击项目卡片 → 弹窗从卡片位置缩放展开，背景半透明遮罩

### 4.2 弹窗内容
| 模块 | 内容 |
|------|------|
| 项目名称 | 产品全称 |
| 一句话定位 | 产品核心价值主张 |
| 产品介绍 | 2-3 段，背景/功能/价值（200-300字） |
| 技术标签 | Python / Streamlit / Claude API / Telegram Bot 等 |
| 操作按钮 | "在线体验" → 项目链接 / "GitHub 仓库" → 源码链接 |

### 4.3 关闭方式
- 点击遮罩层关闭
- 点击右上角 × 关闭
- 按 ESC 关闭

## 五、视觉设计

### 5.1 配色
- 主背景：浅蓝紫 → 浅粉流动渐变（CSS gradient + animation）
- 卡片：rgba(255,255,255,0.6) + backdrop-blur + 微光边框
- 文字主色：深灰 #1a1a2e
- 强调色：蓝紫渐变 #6366f1 → #a855f7
- 标签背景：半透蓝紫

### 5.2 动效清单
| 元素 | 动效 | 触发时机 |
|------|------|---------|
| Hero 文字 | fadeInUp（淡入上移） | 页面加载 |
| 背景渐变 | 缓慢流动旋转 | 持续 |
| 项目卡片 | 上浮 8px + 阴影扩散 | hover |
| 弹窗 | scale(0.95→1) + opacity(0→1) | 打开 |
| 遮罩 | opacity(0→1) | 打开 |

### 5.3 字体规范
- 标题：Inter 700（英文）/ 系统粗体（中文）
- 正文：Inter 400 / 系统常规
- 代码/标签：Inter 500，等宽数字

| 断点 | 布局 |
|------|------|
| ≥1024px | 卡片 3 列并排 |
| 768-1023px | 卡片 2 列 |

## 七、数据来源

项目内容从现有产品的 `产品介绍.md` 提取，不需要 CMS 或数据库。项目卡片和弹窗内容直接硬编码为静态数据，后续维护时手动更新。

---

## 八、不做的事情（YAGNI）

- 不做暗色模式切换
- 不接 CMS 或数据库
- 不做多语言
- 不做博客/文章模块
- 不做访问统计
- 不做项目筛选/搜索
