# 产品更新日志 — 个人AI作品集网站

## v1.1 — 2026-05-27（新增）

### 本次改动核心功能
1. 新增2个项目：琵琶简谱自动生成器（Web应用，Vercel部署）、OfferWin（微信小程序，二维码扫码体验）
2. Hero区个人介绍文案更新：更精准定位「互联网大厂B端AI+产品经理」，强化 Claude Code / Vibe Coding 个人标签
3. ProjectModal 条件渲染：liveUrl/githubUrl/qrcode 三种入口按需显示，小程序类项目展示二维码

### 本次使用AI技术
- 数据模型扩展：Project 接口新增可选的 `qrcode` 字段
- 前端逻辑：弹窗内条件渲染，支持文本链接、URL按钮、二维码图片三种展示形态

### 开发落地难点
1. **小程序项目如何在线展示**：微信小程序无法通过URL直接访问，解决方案是在弹窗中展示小程序官方二维码，用户微信扫码即可体验
2. **非Web项目的链接处理**：OfferWin 无 liveUrl 也无 githubUrl，需要让整个按钮区域在无任何链接时隐藏，而非显示空白按钮

### 解决方案与取舍
- 选择在 Project 接口新增 `qrcode?` 可选字段，而非为小程序单独做一套弹窗模板，理由是保持代码简洁，一行字段即可覆盖新场景
- ProjectModal 采用"二维码优先"逻辑：有 qrcode 显示二维码，否则走原有按钮逻辑，两个分支互不干扰
- 二维码图片放在 `public/` 目录，Next.js 直接静态分发，零额外配置

### 产品复盘思考
本次新增两个项目后，作品集从3个项目扩展到5个，覆盖了Web应用、微信小程序、Telegram Bot三种产品形态。个人介绍文案的迭代也经历了3次微调，最终定位更精确：不仅强调AI产品经理背景，还突出 Claude Code 重度用户和 Vibe Coding 实践者的身份。这个细节让面试官能快速建立「这个人真的在用AI写代码」的认知，比泛泛的「AI Agent开发」更有说服力。

## v1.0 — 2026-05-20（新增）

### 本次改动核心功能
首次上线个人AI作品集网站，包含Hero首屏、3个项目卡片、详情弹窗、响应式布局。

### 本次使用AI技术
- 框架：Next.js 16 + TypeScript
- 样式：Tailwind CSS v4 毛玻璃效果 + 流动渐变背景
- 动效：Framer Motion 淡入上浮 + hover缩放 + 弹窗动画
- 部署：Vercel（GitHub自动部署）

### 开发落地难点
1. **Tailwind v4 兼容适配**：create-next-app 默认安装 Tailwind v4，原计划基于 v3 编写。v4 采用 CSS-based 配置（@theme 指令），不需要 tailwind.config.ts。自定义颜色通过 @theme 变量注入，但 opacity 修饰符（如 text-dark/60）在 v4 中行为不同，部分场景改用 inline style 兜底
2. **毛玻璃效果在 Safari 上的兼容**：需要同时设置 `backdrop-filter` 和 `-webkit-backdrop-filter`
3. **弹窗动画的关闭时机**：Framer Motion 的 AnimatePresence 需要在组件卸载时播放退出动画，使用 `exit` prop + `AnimatePresence` 包裹解决
4. **Google Fonts @import 与 Tailwind v4 CSS 排序冲突**：Tailwind v4 PostCSS 插件将 @import "tailwindcss" 展开为大量 CSS 规则后，Google Fonts 的 @import url(...) 被排在规则之后，违反 CSS 规范。解决方案：将字体加载从 CSS @import 移至 layout.tsx 的 <link> 标签

### 解决方案与取舍
- 选择适配 Tailwind v4 而非降级到 v3，理由是 v4 是当前最新稳定版，CSS-based 配置更简洁，且 Next.js 16 原生集成
- 选择纯静态硬编码数据而非CMS，理由是3个项目的更新频率极低，CMS反而增加复杂度
- 选择单页应用而非多页路由，理由是3张卡片的情况下多页显得空洞，单页+弹窗体验更流畅
- 放弃暗色模式，理由是同一个人使用双套配色增加50%设计和维护成本，暂不必要
- Tailwind v4 opacity 修饰符兼容性问题：对 `text-dark/60` 等写法，改用 React inline style 直接设置颜色值，虽然代码略长但行为确定

### 产品复盘思考
本次是个人第一个纯前端作品集网站。最大收获是意识到：作品集的核心不是技术有多炫，而是让面试官在30秒内看明白你是谁、做了什么。因此设计上刻意控制信息密度——Hero区3行字、卡片3张、弹窗300字，避免信息过载。技术选型上，Next.js + Tailwind v4 + Framer Motion 的组合在开发效率和视觉效果之间取得了很好的平衡。
